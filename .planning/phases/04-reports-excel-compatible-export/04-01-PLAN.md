```php
<?php
// Session guard — idempotent, safe to call multiple times
if (session_status() === PHP_SESSION_NONE) {
    session_start();
}
```

**`getSeedReports()`** — Returns an array of 3 seed report entries. Each entry has keys:
- `nama` (string) — Employee name
- `tahun_bergabung` (int) — Join year
- `total_cuti` (int) — Computed dynamically via `array_sum(array_column(hitungHakCuti($tahun)['data'], 'hari_cuti'))`
- `is_sample` (bool) — `true` for seed data

Use these 3 seeds (matching employee/dashboard.php preset names):
1. Budi Santoso, 2018
2. Siti Rahayu, 2020
3. Ahmad Fauzi, 2022

The function must `require_once __DIR__ . '/cuti-calculator.php'` to call `hitungHakCuti()` for dynamic total computation.

**`initReports()`** — If `$_SESSION['reports']` is not set, populate it with `getSeedReports()`. Always return `$_SESSION['reports']`.

**`getReports()`** — Return `$_SESSION['reports']` or empty array if not set.

**`saveReport($nama, $tahunBergabung)`** — Compute `total_cuti` via `hitungHakCuti()`, build entry with `is_sample => false`. Check for existing entry via `findReportIndex($nama, $tahunBergabung)`. If found, overwrite and return `'updated'`. If not found, append and return `'created'`.

**`findReportIndex($nama, $tahunBergabung)`** — Iterate `$_SESSION['reports']`, match on `nama === $nama && tahun_bergabung === (int)$tahunBergabung`. Return index or `false`.

**`resetReports()`** — `unset($_SESSION['reports'])`. Return nothing.

**`countReports()`** — Return `count($_SESSION['reports'])` or 0.

**`countPresetKaryawan()`** — Return 3 (hardcoded count of seed profiles, matching the seed array length).

Do NOT include any HTML output in this file. Pure PHP functions only.

## Part A: PHP Logic Changes (top of hr/dashboard.php)

**A1. Add reports-data.php include** after the calculator include (line 8):


**A2. Initialize reports** before any form handling:
```php
$reports = initReports();
```

**A3. Handle POST for save action** — Add a POST handler BEFORE the existing GET handler. Pattern: POST-Redirect-GET.

When `$_SERVER['REQUEST_METHOD'] === 'POST'` AND `$_POST['action'] === 'save_report'`:
1. Extract `$tahun_input = trim($_POST['tahun_bergabung'])`, `$nama = trim($_POST['nama_karyawan'])`, `$simpan = isset($_POST['simpan_laporan'])`
2. Validate tahun via `validasiTahunBergabung($tahun_input)`
3. If valid AND `$simpan` is true:
   - Validate `$nama`: must be >= 3 characters. If too short, set flash: `$_SESSION['flash'] = ['type' => 'danger', 'message' => 'Nama karyawan minimal 3 karakter.']`
   - If name valid: call `saveReport($nama, (int)$tahun_input)`. Set flash based on result:
     - `'created'`: `$_SESSION['flash'] = ['type' => 'success', 'message' => "Laporan disimpan untuk $nama ($tahun_input)."]`
     - `'updated'`: `$_SESSION['flash'] = ['type' => 'success', 'message' => "Laporan diperbarui untuk $nama ($tahun_input)."]`
4. If valid but `$simpan` is false: no save, just redirect with year for display
5. If invalid: set flash with error message
6. Redirect: `header('Location: dashboard.php?tahun_bergabung=' . urlencode($tahun_input)); exit;`

When `$_SERVER['REQUEST_METHOD'] === 'POST'` AND `$_POST['action'] === 'reset_reports'`:
1. Call `resetReports()`
2. Re-initialize: `initReports()` (this is optional — could leave empty after reset, but per decision "clears both sample data and any saved entries" → just `resetReports()` without re-init)
3. Actually, per decision: "after reset, stat cards and table should refresh to the empty state message" — so do NOT re-init seeds. Just `resetReports()`.  
4. Set flash: `$_SESSION['flash'] = ['type' => 'info', 'message' => 'Semua laporan telah direset.']`
5. Redirect: `header('Location: dashboard.php'); exit;`

**A4. Flash message handling** — After POST handlers, read and clear flash:
```php
$flash = null;
if (isset($_SESSION['flash'])) {
    $flash = $_SESSION['flash'];
    unset($_SESSION['flash']);
}
```

**A5. Refresh reports after redirect** (for display):
```php
$reports = getReports();
if ($reports === null) $reports = [];
```

**A6. Compute report table helpers:**
- Get current year status for each report by calling `hitungHakCuti($report['tahun_bergabung'])` and finding the row where `tahun_kalender == current year`. Store `status` and `status_class` per report.
- Build a `$report_display` array with enriched entries (adding `current_status` and `current_status_class` fields).
- Handle filter: if `$_GET['filter_status']` is set and not empty, filter `$report_display` to matching status.
